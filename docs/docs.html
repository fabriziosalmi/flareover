Documentation

flareover reads a live Cloudflare zone and rebuilds it — DNS, TLS, redirects, WAF, caching — on EU-sovereign, self-hosted or EU-owned managed infrastructure. It never emits config that silently changes behavior: what it can't map exactly, it asks about or flags. This is the working reference; the landing is the overview and architecture.md the internals.

Install

# Homebrew (macOS/Linux)
brew install fabriziosalmi/flareover/flareover

# or a prebuilt binary
curl -fsSL https://raw.githubusercontent.com/fabriziosalmi/flareover/main/install.sh | sh

# or from source (Go 1.25+, zero external dependencies)
go install github.com/fabriziosalmi/flareover/cmd/flareover@latest

flareover version

The engine is standard-library only — no runtime dependencies, no telemetry, no network access outside the read-only Cloudflare API and your own target's APIs.

The five commands

A migration is read-only until the very last step. Extract, look at the honest report, answer the bounded questions, generate the config, then stand your target up.

# 1. read the live zone (read-only token)
export CLOUDFLARE_API_TOKEN=… CLOUDFLARE_ACCOUNT_ID=…
flareover extract example.com > snap.json

# 2. see exactly what maps, what it must ask, what stays manual
flareover assess snap.json # AUTO · ASK · MANUAL

# 3. answer the bounded yes/no questions → decisions.lock
flareover resolve snap.json > decisions.lock

# 4. generate the target config (Caddyfile, WAF, zone, MIGRATION.md)
flareover prepare snap.json --decisions decisions.lock --edge-ip 5.9.1.1 --out out/

# 5. stand it up on your own target, your own credentials
flareover provision --snapshot snap.json --decisions decisions.lock \
  --pdns-url http://pdns:8081 --certmate-url http://certmate:8000
Nothing is guessed. Steps 1–4 never write to your source or registrar. Only provision (your target) and the DNS flip in execute change anything — and the flip is gated on a live parity check.

The pipeline

A Cloudflare zone goes in; a faithful EU stack comes out; a verdict is attached to every element on the way. Extraction and classification never know which target is in play — the provider-agnostic IR is the seam, so adding a backend is adding one adapter.

extract  →  CF-IR  →  classify  →  report (AUTO·ASK·MANUAL)
                       │
              CF-IR + decisions.lock  →  plan  →  generate
                       │
              provision  →  present (parity gate)  →  execute  →  guard

Classification and generation are a pure function of snapshot + decisions.lock: identical input yields byte-identical config, golden-tested. The report is reviewable in git before anything runs.

The 0% false-positive contract

Every element takes exactly one path. A single interpreter (cfexpr) decides whether something is faithfully translatable, and it is used by both the classifier and the generator — so they can never disagree. A rule is AUTO only when the generator actually emits config for it.

VerdictMeaningWhat happens
AUTOProvably-equivalent target mapping.Config generated, 1:1.
ASKBounded, enumerable ambiguity.One yes/no; emitted only if answered.
MANUALNo faithful deterministic mapping.Documented with the exact params, never guessed.

The invariant is symmetry: classify ⟺ generate. If the generator would drop something, the classifier must not call it AUTO. A host/path-scoped transform, a compound WAF expression, a Cloudflare challenge with no simple match — each is surfaced honestly, never a hopeful AUTO. Determinism is enforced too: no wall-clock, no randomness, no network in classify/generate.

Sovereignty tiers

An EU region is not EU sovereignty. A US-operated cloud in Milan gives European data residency but stays reachable under the US CLOUD Act. flareover tags every target with its operator and states the trade-off — it never markets it away.

TierWho controls itExamples
EU-ownedEU-headquartered operator; no non-EU compulsion path.Hetzner, OVHcloud, Scaleway, Aruba, Leaseweb, IONOS, Contabo, UpCloud, bunny.net, Gandi
EU-adjacentNon-EU jurisdiction, full GDPR (stated as such).Infomaniak (CH)
US-operator / EU-regionEU data residency, but US CLOUD Act / FISA reach.AWS · GCP · Azure EU regions

Run flareover providers for the live catalogue. Corporate-jurisdiction information to inform a choice — confirm the exact region and terms with the provider at deploy time.

CLI — every phase

Twelve verbs walk a migration end to end. Everything up to execute is read-only against your source.

PhaseWhat it does
zonesList every zone the token can see (account-scoped read migrates any/all of them).
extractRead a live zone (read-only API) into a snapshot JSON. Needs CLOUDFLARE_API_TOKEN.
assessClassify the snapshot into an honest AUTO/ASK/MANUAL report. --md / --json / --html.
resolveWalk the ASK questions into a decisions.lock (interactive, or --defaults / --merge).
costEstimate the Cloudflare tier/add-on cost vs a flat EU stack (--vps to override).
prepareGenerate the target artifacts + a MIGRATION.md report. --edge-ip, --dns, --stack, --validate, --edge-provider, --mesh-edge.
provisionStand the target up via APIs: DNS zone + DNSSEC, CertMate DNS-01. --pdns-url or --dns scaleway|ovh|gandi|leaseweb|hetzner|route53|clouddns|azure.
presentParity gate: probe the live edge vs the staged edge (--after-addr). PASS/FAIL.
executeOrchestrate the phases live up to the gated cutover; the DNS flip stays your explicit step.
guardFailguards watchdog: health-watch + rollback/failover trigger (--on-unhealthy, --interval).
storageMigrate object storage (R2/S3) → MinIO or managed EU S3 (--dest scaleway|ovh|contabo|aruba) + an rclone plan.
doctorRead-only pre-flight: is every target reachable, authorized, configured? GO/NO-GO.
providersList EU edge providers with their honest sovereignty tier.

The rest of the stack

CaddyReverse proxy · native ACME TLS · HTTP/3. The flagship proxy profile.
caddy-wafWAF: OWASP, rate-limit, IP·ASN·country blocking, external blocklists.
souinEdge cache (Caddy module).
CertMateCertificates via DNS-01: wildcard, Let's Encrypt or Actalis (EU CA).
MinIO / EU S3Object storage target — self-hosted MinIO or managed Scaleway/OVH/Contabo/Aruba; rclone copies the data.
WireGuard meshKeep an unchanged origin: the edge dials in over an encrypted tunnel; origin has zero public inbound.
Managed DNS/storage/edge adapters are unit- and wire-verified against each vendor's API; run docs/live-proof.md to promote them to proven-live before a real migration. DNSSEC on a managed provider is surfaced but not yet automated — never silently assumed.

Cookbook — Cloudflare → self-hosted EU

The full-sovereignty path: your own PowerDNS + Caddy on an EU VPS.

flareover extract example.com > snap.json
flareover resolve snap.json --defaults > decisions.lock
flareover prepare snap.json --decisions decisions.lock --edge-ip $EDGE_IP --out out/
flareover doctor --pdns-url http://pdns:8081 --certmate-url http://certmate:8000  # GO/NO-GO
flareover provision --snapshot snap.json --decisions decisions.lock \
  --pdns-url http://pdns:8081 --pdns-key $PDNS_KEY \
  --nameservers ns1.example.eu,ns2.example.eu \
  --certmate-url http://certmate:8000 --certmate-token $CM --ca actalis
flareover present --after-addr $EDGE_IP   # parity gate before you flip DNS
No box to run the stack on? deploy/ is a one-command docker compose landing zone (PowerDNS · Caddy+caddy-waf+souin · CertMate · MinIO) that consumes the prepare --out artifacts directly.

Cookbook — Cloudflare → managed EU DNS

Prefer managed over self-hosting the nameservers? Point --dns at an EU-owned provider; the records map deterministically and the apply is idempotent.

flareover prepare snap.json --decisions decisions.lock --dns scaleway --out out/  # preview the zone

export SCW_SECRET_KEY=… SCW_DEFAULT_PROJECT_ID=…
flareover provision --snapshot snap.json --decisions decisions.lock --dns scaleway
# → prints the delegation nameservers; set them at your registrar, then let TTLs expire.

Re-running provision is the idempotency check: a correct adapter converges with no duplicate records. Swap scaleway for another EU-owned provider — ovh, gandi, leaseweb, hetzner, or bunny (preview + apply.sh) — or, honestly tiered as US-operated, route53 / clouddns / azure.

Cookbook — keep your origin (WireGuard mesh)

The lowest-risk shape leaves your origin exactly where it is and just re-tunnels it: swap cloudflared for WireGuard. flareover can also emit a create script that boots the edge on Scaleway or OVH from the generated cloud-init, or a terraform/hetzner/ module that boots it on Hetzner.

flareover prepare snap.json --decisions decisions.lock \
  --mesh-edge scaleway=$ORIGIN_IP:51820 --edge-provider scaleway --out out/

export SCW_SECRET_KEY=… SCW_DEFAULT_PROJECT_ID=…
sh out/edge/create-scaleway.scaleway.sh   # boots the edge, runs the cloud-init, prints its IP
flareover prepare snap.json --decisions decisions.lock --edge-ip <that-ip> --out out/

The edge builds Caddy (caddy-waf + souin), brings up the mesh, and fails closed on the firewall. The origin keeps zero public inbound.
